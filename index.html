CSS Animation

 One of the many interesting capabilities of CSS is its ability to animate objects within itself. This is particularly useful when you want to add a little bit of interesting elements to your otherwise simple website. Not only does it make the website look better it’s also got a few other benefits on it’s side. The first is that it is so simple it doesn’t even require an outside coding language like Java to work. Next Even with a moderate load on your system the animation will still run beautifully and won’t skip frames during it. Finally, its run entirely within the browser meaning it’s more efficient and optimized allowing for it to run properly each time. This is a great way to improve your website without too much of a headache. Let’s get into the important functions needed to make the animation work.

 To understand CSS animation, you have to understand two key components that make the animation work. The first of these two components is the animation property. The animation property is used to set the timing of the animation so that it occurs during a set period of time. It also allows you to delay the animation set which direction it travels and how fast it moves between frames. It uses the following sub-properties.

  • animation-delay:  Specifies the delay between an element loading and the start of an animation sequence and whether the animation should start immediately from its beginning or partway through the animation.
  • animation-direction:  Specifies whether an animation's first iteration should be forward or backward and whether subsequent iterations should alternate direction on each run through the sequence or reset to the start point and repeat.
  • animation-duration:  Specifies the length of time in which an animation completes one cycle.
  • animation-fill-mode:  Specifies how an animation applies styles to its target before and after it runs.
  • animation-iteration-count:  Specifies the number of times an animation should repeat.
  • animation-name:  Specifies the name of the @keyframes at-rule describing an animation's keyframes.
  • animation-play-state:  Specifies whether to pause or play an animation sequence.
  • animation-timing-function:  Specifies how an animation transitions through keyframes by establishing acceleration curves.

 Speaking of frames this brings us to the second most important feature of an animation the @keyframes rule. This rule is what makes the animation look like an animation meaning it renders the animation at certain frames to make it look animated. It does this through a percentage tag. This percentage tells us how far through the time of the frame before it changes the frame to a different one to animate it. This will make an element in the code appear like its spinning or sliding in accordance with the what percent of the way through an animation time the keyframe is.
